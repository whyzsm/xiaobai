# Frontend Delivery State

## 中文

## 当前状态

- Loop：frontend-delivery
- 状态：ready-for-manual-run
- 当前阶段：需求接入与前端技术设计门禁

## 运行规则

- 需求来源支持本地文档、对话输入和语雀链接/API。
- 主设计文档必须写入主目标仓，各仓补充分设计文档必须写入各自目标仓。
- 独立 evaluator 通过设计评审后，仍必须等待 `human-design-approval`，之后才允许进入编码。
- 工程仓只记录状态、门禁结果、源链接、目标仓和 PR 链接，不保存业务设计正文或业务代码。

## English

## Current State

- Loop: frontend-delivery
- Status: ready-for-manual-run
- Current stage: requirement intake and frontend technical design gate

## Run Rules

- Requirement sources include local documents, conversation input, and Yuque links/API.
- The master design document must be written to the primary target repository, and repository supplements must be written to each participating target repository.
- Even after independent evaluator design approval, implementation must wait for `human-design-approval`.
- The engineering repository records only state, gate results, source links, target repositories, and PR links. It must not store business design bodies or business code.
