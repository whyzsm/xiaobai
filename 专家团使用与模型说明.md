## 怎么一次性跑完整评审 / How to run a full review in one go

1. 专家中心打开「项目评审团」（`loop-review-team`）。
   Open the "Project Review Panel" (`loop-review-team`) in the Expert Center.
2. 发送「评审当前项目（完整流程）」（或点快捷提示）。主理人罗统筹会：
   Send "Review the current project (full workflow)". Lead **罗统筹** will:
   - **Phase 0** 亲自 `TeamCreate` 建队（团队名 `loop-review-team`）。
   - **Phase 1（并行）** 调度三位成员独立评审：甄明察（Agent 工作流）、高见远（架构）、严析弊（批判）。
   - **Phase 2–4** 收齐回传 → 识别共识/分歧 → 必要时深化 → 汇编综合报告（含共识、分歧裁决、优先级行动 A1–A7）。
3. 也可点「重点审 agent 工作流与评审门禁」或「重点挑架构与过度设计风险」走单维深入流程。
   Or use the quick prompts for a single-dimension deep dive.

## 重要模型限制（务必知悉）/ Critical model limitation

- 专家包 `plugin.json` **不含 `modelId` 字段**——运行模型由专家中心在发起对话时选定。
  The `plugin.json` has **no `modelId` field** — the model is chosen when you start the chat in the Expert Center.
- Team 运行时，主理人与各成员在**同一会话**内协作，成员通常**继承主对话所选模型**，不一定各自用不同模型。
  In Team mode the lead and members collaborate in one session; members usually **inherit the main chat's model**, so they may not each run a different model.
- 若要**精确还原"每成员不同模型"**的设定，请用原 4 个独立 Agent 专家，在专家中心分别选
  Deepseek-V4-Pro / Hy3 / GLM-5.2 / Kimi-K2.7-Code 运行。
  To **exactly reproduce per-member models**, use the 4 original standalone Agent experts and pick the assigned model per expert.
- 每位成员的 Agent 定义里已把"应使用某模型"写入**身份声明**作为强约束。
  Each member's Agent definition states its assigned model as a hard identity constraint.

## 与原 4 个独立专家的关系 / Relation to the 4 standalone experts

- 原 4 个独立专家（loop-review-coordinator / loop-agent-reviewer / loop-architect-reviewer / loop-critical-reviewer）**仍保留**，用于单独深挖某一维度或精确分模型运行。
  The 4 original standalone experts remain available for single-dimension deep dives or exact per-model runs.
- 是否删除原 4 个以消重，由用户在对话中确认。
  Whether to delete the 4 originals to avoid duplication is confirmed with the user.
